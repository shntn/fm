# テスト

## 共通

- 実装の詳細ではなく、動作を検証するテストを作成する
- 各テストには明確なアサーションが1つ必要です。検証する内容に基づいて名前を付けてください
- モジュールの内部関数ではなく、公開APIをテストしてください
- モックよりも実際の依存関係を優先してください
  - Luaのテスト: Rust APIをモックして Lua ロジックを単体テスト
  - Rustのテスト: Lua実行環境をモックして Rust インターフェース層を単体テスト
- すべてのバグ修正には、修正を適用しないと失敗する回帰テストを含める必要があります
- タスクを完了とマークする前に、テストスイート全体を実行してください
  - `cargo test`
  - `busted tests/lua/`

## Lua

- 関連するテストケースをグループ化するには、`describe` ブロックを使用します
- 個々のテストケースには、`it` を使用します

## Rust

- 関連するテストケースをグループ化するには、`mod` ブロックを使用します
- 個々のテストケースには `fn` を使用します
