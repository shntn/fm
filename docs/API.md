
# Rust インタフェース

## 2.1 ターミナル制御（screen.*）

### `screen.clear()`

ターミナル画面全体をクリアし、カーソルを(0,0)に移動する。

- 引数: なし
- 戻り値: なし
- エラー時: 何もしない（panicしない）

### `screen.write(x, y, text)`

(x, y) 座標に文字列を描画する。左上を (0,0) とする。

- 引数: `x, y`（描画位置の座標）, `text`（描画する文字列）
- 戻り値: なし
- 範囲外: 指定座標がターミナルの範囲外の場合、その呼び出しを無視する（エラーにしない）
- 改行: text 内の改行文字は描画せず、何も表示しない（Lua側で改行を含む文字列を渡した場合は、改行以降が欠落する。この挙動でよい）
- マルチバイト対応: UTF-8文字の幅は考慮しなくてよい（1文字=1幅として扱う。厳密なマルチバイト対応は後日）

### `screen.get_size()` → width, height

現在のターミナルの列数（width）と行数（height）を整数で返す。

- 引数: なし
- 戻り値: `width`（列数）, `height`（行数）
- エラー時: (80, 24) を返す

## 2.2 ファイルシステム操作（fs.*）

### `fs.list(path)` → files_table | nil, error_message

指定されたパスのディレクトリ内容を読み取り、テーブルを返す。

- 引数: `path`（読み取り対象のディレクトリパス）
- 戻り値（成功時）: files_table
- 戻り値（失敗時）: `nil, error_message`
- エラー条件: パスが存在しない、またはディレクトリでない場合
- hiddenファイル: `.`で始まるファイルも含め、すべてのエントリを返す。フィルタリングはLua側の責務
- 特殊ファイルシステム: Linuxの `/proc`, `/sys` のような特殊ファイルシステム上のディレクトリを読む際のエラーもls相当の挙動でよい
- macOSの `.DS_Store` のような隠しファイルも含まれる

files_table の各要素の形式:

```lua
{
    name     = "README.md",      -- 文字列: ファイル名
    is_dir   = false,            -- 真偽値: ディレクトリか
    size     = 12980,            -- 数値: バイト数
    modified = "2025-06-27 10:00:00",  -- 文字列: YYYY-MM-DD HH:MM:SS 形式
    perm     = "rw-r--r--",      -- 文字列: Unixパーミッション表記
}
```

### `fs.run(command)` → exit_code

シェル経由で外部コマンドを実行する（`/bin/sh -c command`）。

- 引数: `command`（実行するコマンド文字列）
- 戻り値: `exit_code`（コマンドの終了コード）
- 標準出力・標準エラー出力: ターミナルにそのまま表示される（フレームワーク側でキャプチャしない）
- 実行方式: コマンド実行中、フレームワークは待機する（同期的な呼び出し）

### `fs.cwd()` → path

現在のワーキングディレクトリの絶対パスを文字列で返す。

- 引数: なし
- 戻り値: `path`（現在のワーキングディレクトリの絶対パス）

### `fs.read_file(path)` → content | nil, error_message

指定されたパスのファイル内容を文字列として読み取る。

- 引数: `path`（読み取り対象のファイルパス）
- 戻り値（成功時）: `content`（ファイル内容の文字列）
- 戻り値（失敗時）: `nil, error_message`
- エラー条件: パスが存在しない、または読み取れない場合

## 2.3 TOMLパース（toml.*）

### `toml.parse(text)` → table | nil, error_message

TOML形式の文字列をLuaテーブルに変換する。

- 引数: `text`（TOML形式の文字列）
- 戻り値（成功時）: `table`（パース済みの連想配列。セクションはネストしたテーブルになる）
- 戻り値（失敗時）: `nil, error_message`
- エラー条件: TOMLとして構文が不正な場合

## 2.4 キー入力の変換

端末から読み取った生バイト列を、以下のキー名に変換してLuaに渡す。

v0.1で対応するキー一覧:

| キー名 | 対応する物理キー |
|--------|-----------------|
| `"up"` | ↑ |
| `"down"` | ↓ |
| `"left"` | ← |
| `"right"` | → |
| `"enter"` | Return / Enter |
| `"tab"` | Tab |
| `"backspace"` | Backspace |
| `"space"` | Space |
| `"escape"` | Esc |
| `"home"` | Home |
| `"end"` | End |
| `"delete"` | Delete（Forward Delete） |

印字可能文字:

上記に該当しない1バイトの印字可能文字（英字 `a-z A-Z`、数字 `0-9`、記号）は、そのまま1文字の文字列として渡す（例: `"a"`, `"3"`, `"/"`, `"."`）。

キー変換の方針:

- エスケープシーケンスのパースはテーブル駆動で実装する。キーを追加する際はテーブルにエントリを1行追加するだけで済む構造とする
- 未知のエスケープシーケンスは無視する（Luaに渡さない）
- macOS Terminal.app と iTerm2、Linuxの主要なターミナルエミュレータ（xterm, gnome-terminal, alacritty）で動作確認すること

# Lua スクリプト

## 3.1 コールバック定義

Luaスクリプト側が以下の2つの関数を定義する。

### `on_init()`

フレームワーク起動時に1回だけ呼ばれる。

- 引数: なし
- 戻り値: 無視される
- 用途: ここでカレントディレクトリの読み込みや初期描画を行う

### `on_key(key)`

キーが押されるたびに呼ばれる。

- 引数: `key`（「キー入力の変換」のキー名）
- 戻り値: `false` を返すと、フレームワークがメインループを終了し、ターミナルを復元してプログラムを終了する
- 戻り値: `true` または `nil` を返すと、フレームワークは次のキー入力を待つ

## 3.2 テンプレートエンジン（template.*）

Lua側で実装するモジュール。`require("template")` で読み込む。

### `template.render(tmpl, vars)` → result

テンプレート文字列と連想配列を受け取り、プレースホルダを展開した文字列を返す。

- 引数: `tmpl`（プレースホルダを含むテンプレート文字列）, `vars`（キーと値の連想配列。値はすべて文字列。呼び出し側で文字列化済み）
- 戻り値: `result`（プレースホルダを展開済みの文字列）
- 未定義キー: `vars` にキーが存在しない場合は空文字列に展開する
- 幅N: バイト数や文字数ではなく、`utf8width`モジュールによる画面上の表示幅（半角=1、全角=2）を基準にパディングする
- 幅超過: 値の表示幅が幅Nを超える場合は切り詰める。左詰め（`{key:N}`・`{key:N:left}`）は末尾を、右詰め（`{key:N:right}`）は先頭を切り詰める（省略記号は付けない）

プレースホルダ書式:

| 書式 | 内容 |
|---|---|
| `{key}` | 値をそのまま展開する |
| `{key:N}` | 幅Nで左詰め展開する |
| `{key:N:right}` | 幅Nで右詰め展開する |
| `{key:N:left}` | 幅Nで左詰め展開する |

使用例:

```lua
local t = require("template")

local s = t.render("{name:20}  {size:8:right}  {date}", {
    name = "README.md",
    size = "1234",
    date = "2025-06-27 10:00:00",
})
-- → "README.md             1234  2025-06-27 10:00:00"
```

## 3.3 レイアウトエンジン（layout.*）

アプリケーション層（ファイルマネージャ）で実装するモジュール。画面レイアウトを定義するテンプレート文字列を、`template.render` に渡せる文字列の配列に展開する。

### `layout.expand(tmpl, columns, width, height)` → lines

ユーザー定義のレイアウトテンプレートを、現在の画面サイズに応じてテンプレートエンジン向けの文字列の配列に展開する。

- 引数: `tmpl`（ユーザー定義のテンプレート文字列）, `columns`（カラム定義の連想配列）, `width`（現在の画面幅）, `height`（現在の画面高さ）
- 戻り値: `lines`（テンプレートエンジンに渡せる文字列の配列。要素数は `height` と一致する）
- 制約: 繰り返し行は1テンプレートに1行まで。複数指定はエラー
- 文字幅: 1文字=1幅として扱う（UTF-8全角対応は後日）
- 行頭マーカー: 出力文字列に含まない

テンプレート記法:

| 項目 | 内容 |
|---|---|
| 行頭マーカー | スペース=通常行、`@`=繰り返し行 |
| タグ記法 | `#name#####` の形式。先頭の `#` から末尾の `#` までの合計文字数が幅 |
| 基準サイズ | 80x25 |
| flex幅計算（1タグ） | `基準幅 + (現在画面幅 - 80)` |
| flex幅計算（複数タグ） | `基準幅 + floor((現在画面幅 - 80) ÷ flex タグ数)` |
| 繰り返し行 | 1テンプレートに1行まで |
| 繰り返し件数 | `画面高さ - 通常行数` で自動計算 |
| 同名タグ複数 | 配列グループとして認識 |

columns定義:

| キー | 型 | 必須 | 内容 |
|---|---|---|---|
| `field` | string | 必須 | データのパス。`[]`を含む場合は配列グループ |
| `flex` | boolean | 任意 | trueのとき画面幅に応じて伸縮 |
| `align` | string | 任意 | `"left"`（デフォルト）または `"right"` |
| `dir` | string | 任意 | `"vertical"`または`"horizontal"`。配列グループでのみ意味を持ち、省略時は`"horizontal"` |
| `before` | boolean | 任意 | trueのとき、このタグの直前に装飾用プレースホルダを生成する |
| `after` | boolean | 任意 | trueのとき、このタグの直後に装飾用プレースホルダを生成する |

### `layout.tag_width(tmpl, key)` → width

テンプレート文字列内で、指定したキーのタグの幅（`#`の合計文字数）を返す。

- 引数: `tmpl`（テンプレート文字列）, `key`（タグのキー）
- 戻り値: `width`（そのタグの幅）。該当するキーのタグが存在しない場合は`nil`
- 用途: タグの幅をテンプレート外で別の数値として二重管理せず、テンプレート自体から読み出すためのもの（例: `view.lua`がファイル名の切り詰め幅を、テンプレートの`name`タグの幅からそのまま取得する）

### カーソル位置などの装飾（before/after）

ファイル名を反転表示するなど、特定の1行だけを装飾したい場合の仕組み。`layout.expand`自体はカーソル位置を一切知らず、装飾内容の決定は呼び出し側（`fm.lua`）の`vars`構築時に委ねる。

`columns[key].before`または`columns[key].after`が`true`のカラムは、出力テンプレートにおいてタグの前後に幅指定なしのプレースホルダが追加される。

| field の記述 | 生成される出力（before/afterなし） | 生成される出力（before/afterあり） |
|---|---|---|
| `"files[].name"`（配列グループ、`before=true`） | `{files[i].name:N}` | `{files[i].before}{files[i].name:N}` |
| `"files[].size"`（配列グループ、`after=true`） | `{files[i].size:N}` | `{files[i].size:N}{files[i].after}` |
| 非配列カラム（`before=true`） | `{field:N}` | `{before}{field:N}` |

- プレースホルダ名は固定で `before` / `after`。配列グループの場合は他のフィールドと同じインデックス `i` を使う（`{files[i].before}`）
- 幅指定を持たないため、`template.render`のパディング計算には一切影響しない
- `vars`にキーが存在しない場合は空文字列に展開される（`template.render`の既存仕様）ため、装飾しない行では`vars`に何も設定しなくてよい
- `fm.lua`側は、カーソル行に対応するインデックスの`vars["files[i].before"]`にのみエスケープシーケンスなどの文字列を設定すればよい

使用例:

```lua
columns = {
    name = { field = "files[].name", flex = true, dir = "vertical", before = true },
    size = { field = "files[].size", align = "right", dir = "vertical", after = true },
}

-- fm.lua の build_vars 内、カーソル行（例: files[3]）のみ装飾する場合
vars["files[3].before"] = "\27[7m"
vars["files[3].after"] = "\27[0m"
```

出力テンプレート（一部）:

```
{files[3].before}{files[3].name:40}  {files[3].size:6:right}{files[3].after}
```

`template.render`適用後、カーソル行のファイル名とサイズ（間のスペースを含む）が反転表示される。

列の対応付け:

同名タグが1行内に複数回出現する場合、行を左から走査した出現順（1回目=列0、2回目=列1、…）で列番号 `c` を決定する。配列グループに属するすべての行（通常行・繰り返し行）で、このタグの出現順が一致している必要がある。

インデックス計算:

配列グループに属する行の総数（同名タグを含む通常行数 + 繰り返し件数）を `R`、同名タグの列数を `C`、配列グループ内での行番号を `r`（0始まり。通常行→繰り返し行の順に連番）、列番号を `c`（0始まり）とする。

| dir | インデックス計算式 |
|---|---|
| `vertical` | `r + c × R` |
| `horizontal` | `r × C + c` |

出力テンプレートの仕様:

出力はテンプレートエンジンに渡せる文字列の配列。各要素が画面の1行に対応する。

変換ルール:

| 入力 | 出力 |
|---|---|
| 行頭マーカー（スペース・`@`） | 除去される |
| タグ `#name###`（幅N、alignなし） | `{field:N}` |
| タグ `#name###`（幅N、align=right） | `{field:N:right}` |
| タグ `#name###`（flex、幅N） | `{field:M}`（M=flex計算済み幅） |
| 配列グループのタグ | `{field[i].key:N}`（iはインデックス計算式で決定） |
| タグ以外のリテラル文字 | そのまま出力 |
| 通常行 | 1行をそのまま変換して1要素 |
| 繰り返し行 | 繰り返し件数分に展開して複数要素 |

field のパス解決:

| field の記述 | 出力上の表現 |
|---|---|
| `"cwd"` | `{cwd}` |
| `"files[].name"` | `{files[i].name:N}`（`[]`にインデックスを埋め込む） |

出力配列の構造:

- 要素数は `height` と一致する
- 通常行は定義順に配置される
- 繰り返し行は繰り返し件数分展開されて通常行の間に配置される

制約:

- flex タグが複数ある場合、追加幅の余りは無視（切り捨て）
- 繰り返し行は1テンプレートに1行まで。複数指定はエラー
- 配列グループ内で同名タグの出現順が行によって異なる場合はエラー
- 文字幅は1文字=1幅（UTF-8全角対応は後日）
- 行頭マーカーは出力文字列に含まない
