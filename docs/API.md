
# Rust インタフェース

## 2.1 ターミナル制御（screen.*）

### `screen.clear()`

ターミナル画面全体をクリアし、カーソルを(0,0)に移動する。

- 引数: なし
- 戻り値: なし
- エラー時: 何もしない（panicしない）

### `screen.write(x, y, text)`

(x, y) 座標に文字列を描画する。左上を (0,0) とする。

- 引数: `x, y`（描画位置の座標）, `text`（描画する文字列）
- 戻り値: なし
- 範囲外: 指定座標がターミナルの範囲外の場合、その呼び出しを無視する（エラーにしない）
- 改行: text 内の改行文字は描画せず、何も表示しない（Lua側で改行を含む文字列を渡した場合は、改行以降が欠落する。この挙動でよい）
- マルチバイト対応: UTF-8文字の幅は考慮しなくてよい（1文字=1幅として扱う。厳密なマルチバイト対応は後日）

### `screen.get_size()` → width, height

現在のターミナルの列数（width）と行数（height）を整数で返す。

- 引数: なし
- 戻り値: `width`（列数）, `height`（行数）
- エラー時: (80, 24) を返す

## 2.2 ファイルシステム操作（fs.*）

### `fs.list(path)` → files_table | nil, error_message

指定されたパスのディレクトリ内容を読み取り、テーブルを返す。

- 引数: `path`（読み取り対象のディレクトリパス）
- 戻り値（成功時）: files_table
- 戻り値（失敗時）: `nil, error_message`
- エラー条件: パスが存在しない、またはディレクトリでない場合
- hiddenファイル: `.`で始まるファイルも含め、すべてのエントリを返す。フィルタリングはLua側の責務
- 特殊ファイルシステム: Linuxの `/proc`, `/sys` のような特殊ファイルシステム上のディレクトリを読む際のエラーもls相当の挙動でよい
- macOSの `.DS_Store` のような隠しファイルも含まれる

files_table の各要素の形式:

```lua
{
    name     = "README.md",      -- 文字列: ファイル名
    is_dir   = false,            -- 真偽値: ディレクトリか
    size     = 12980,            -- 数値: バイト数
    modified = "2025-06-27 10:00:00",  -- 文字列: YYYY-MM-DD HH:MM:SS 形式
    perm     = "rw-r--r--",      -- 文字列: Unixパーミッション表記
}
```

### `fs.run(command)` → exit_code

シェル経由で外部コマンドを実行する（`/bin/sh -c command`）。

- 引数: `command`（実行するコマンド文字列）
- 戻り値: `exit_code`（コマンドの終了コード）
- 標準出力・標準エラー出力: ターミナルにそのまま表示される（フレームワーク側でキャプチャしない）
- 実行方式: コマンド実行中、フレームワークは待機する（同期的な呼び出し）

### `fs.cwd()` → path

現在のワーキングディレクトリの絶対パスを文字列で返す。

- 引数: なし
- 戻り値: `path`（現在のワーキングディレクトリの絶対パス）

## 2.3 キー入力の変換

端末から読み取った生バイト列を、以下のキー名に変換してLuaに渡す。

v0.1で対応するキー一覧:

| キー名 | 対応する物理キー |
|--------|-----------------|
| `"up"` | ↑ |
| `"down"` | ↓ |
| `"left"` | ← |
| `"right"` | → |
| `"enter"` | Return / Enter |
| `"tab"` | Tab |
| `"backspace"` | Backspace |
| `"space"` | Space |
| `"escape"` | Esc |
| `"home"` | Home |
| `"end"` | End |
| `"delete"` | Delete（Forward Delete） |

印字可能文字:

上記に該当しない1バイトの印字可能文字（英字 `a-z A-Z`、数字 `0-9`、記号）は、そのまま1文字の文字列として渡す（例: `"a"`, `"3"`, `"/"`, `"."`）。

キー変換の方針:

- エスケープシーケンスのパースはテーブル駆動で実装する。キーを追加する際はテーブルにエントリを1行追加するだけで済む構造とする
- 未知のエスケープシーケンスは無視する（Luaに渡さない）
- macOS Terminal.app と iTerm2、Linuxの主要なターミナルエミュレータ（xterm, gnome-terminal, alacritty）で動作確認すること

# Lua スクリプト

## 3.1 コールバック定義

Luaスクリプト側が以下の2つの関数を定義する。

### `on_init()`

フレームワーク起動時に1回だけ呼ばれる。

- 引数: なし
- 戻り値: 無視される
- 用途: ここでカレントディレクトリの読み込みや初期描画を行う

### `on_key(key)`

キーが押されるたびに呼ばれる。

- 引数: `key`（「キー入力の変換」のキー名）
- 戻り値: `false` を返すと、フレームワークがメインループを終了し、ターミナルを復元してプログラムを終了する
- 戻り値: `true` または `nil` を返すと、フレームワークは次のキー入力を待つ

## 3.2 テンプレートエンジン（template.*）

Lua側で実装するモジュール。`require("template")` で読み込む。

### `template.render(tmpl, vars)` → result

テンプレート文字列と連想配列を受け取り、プレースホルダを展開した文字列を返す。

- 引数: `tmpl`（プレースホルダを含むテンプレート文字列）, `vars`（キーと値の連想配列。値はすべて文字列。呼び出し側で文字列化済み）
- 戻り値: `result`（プレースホルダを展開済みの文字列）
- 未定義キー: `vars` にキーが存在しない場合は空文字列に展開する

プレースホルダ書式:

| 書式 | 内容 |
|---|---|
| `{key}` | 値をそのまま展開する |
| `{key:N}` | 幅Nで左詰め展開する |
| `{key:N:right}` | 幅Nで右詰め展開する |
| `{key:N:left}` | 幅Nで左詰め展開する |

使用例:

```lua
local t = require("template")

local s = t.render("{name:20}  {size:8:right}  {date}", {
    name = "README.md",
    size = "1234",
    date = "2025-06-27 10:00:00",
})
-- → "README.md             1234  2025-06-27 10:00:00"
```
