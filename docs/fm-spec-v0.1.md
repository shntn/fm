# 1. プロジェクト概要

## 1.1 概要

ターミナルで動作するファイルマネージャ。
アプリケーションのロジックを Lua に持つことにより、試作・変更を容易にする。

## 1.2 全体アーキテクチャ

- Rust と Lua スクリプトの 2 層構造
- Rust は OS と Lua スクリプトの間を取り持つ薄いインタフェース層
- Lua スクリプト は ファイルマネージャとしてのロジックを持つアプリケーション層

## 1.3 ターゲット環境

- 対応OS : macOS, Linux（クロスプラットフォーム）
- 必須ランタイム: Lua 5.4（mlua経由で静的リンク、またはシステムLuaを使用）
- ビルドツール: Cargo (Rust 1.70+)

## 1.4 開発言語と依存関係

| 言語/ライブラリ | 用途                      | バージョン目安 |
|-----------------|---------------------------|----------------------------|
| Rust            | フレームワーク本体        | 1.70+（edition 2021）      |
| mlua            | Luaランタイム埋め込み     | 0.10+（feature = "lua54"） |
| crossterm       | ターミナル制御            | 0.28+                      |
| 標準ライブラリ  | ファイルI/O、プロセス実行 | Rust標準                   |


# 2. Rustフレームワークの責務

- ターミナル制御 (raw mode, 画面クリア, 描画)
- キー入力取得とキー名への変換
- ファイルシステム操作 (list, run, cwd)
- Luaランタイム (mlua) の埋め込みと実行
- メインイベントループ

# 3. Luaスクリプトの責務

- 状態管理（カレントディレクトリ、カーソル、マーク状態
- キーマッピング
- 画面描画ロジック
- テンプレート展開
- ファイル操作の判断

# 4. ファイル構成

```
fm/
├─ Cargo.toml
├─ src/                 # Rust のソースコード
├─ lua/                 # Lua のソースコード
├─ tests/               # テストコード
│   ├─ lua/            # Lua のテストコード
│   └─ rust/           # Rust のテストコード
└─ docs/                # ドキュメント
```
